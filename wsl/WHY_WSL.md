# 🤔 WSL Proof Server를 사용하는 이유

## 현재 상황

### ✅ 현재 작동하는 방식
- **브라우저 Web Worker** (`src/lib/proof.worker.ts`)에서 직접 증명 생성
- 사용자의 브라우저에서 `snarkjs`를 사용하여 ZKP 증명 생성
- **소요 시간**: 약 15-20초 (기기 성능에 따라 다름)

### ❓ WSL Proof Server의 목적
- **느린 기기(노트북)를 위한 오프로딩**
- 서버에서 증명을 생성하여 브라우저 부하 감소
- **예상 소요 시간**: 3-6초 (서버 성능이 더 좋은 경우)

---

## 🎯 실제로 필요한가?

### ✅ **필요한 경우**
1. **시연 환경**에서 속도가 중요할 때
   - 브라우저에서 15-20초 걸리는 것을 3-6초로 단축
   - 시연 시 더 부드러운 경험 제공

2. **느린 기기 사용자**를 위한 옵션
   - 저사양 노트북/태블릿에서 브라우저 증명 생성이 너무 느릴 때
   - 서버로 오프로딩하여 사용자 경험 개선

3. **라즈베리파이 시연**을 위한 대체
   - 원래 계획은 라즈베리파이에서 증명 서버 실행
   - WSL로 라즈베리파이 역할 대체

### ❌ **필요 없는 경우**
1. **현재 시스템이 잘 작동하는 경우**
   - 브라우저 Web Worker로 충분히 작동
   - 대부분의 사용자 기기에서 15-20초면 충분히 빠름

2. **추가 인프라 관리가 부담스러운 경우**
   - WSL 설정, 서버 실행, 유지보수 필요
   - 단순히 브라우저에서 작동하는 것이 더 간단

3. **시연이 아닌 실제 사용 환경**
   - 일반 사용자는 브라우저에서 직접 증명 생성해도 문제없음
   - 추가 서버 인프라 불필요

---

## 💡 결론

### **WSL Proof Server는 선택 사항입니다**

현재 시스템은 **브라우저 Web Worker만으로도 완전히 작동**합니다.

WSL Proof Server를 사용하려면:
1. 프론트엔드 코드 수정 필요 (WSL 서버 호출 로직 추가)
2. WSL 환경 설정 및 서버 실행 필요
3. 네트워크 연결 필요 (localhost:8787)

**권장 사항**:
- **시연 시**: WSL Proof Server 사용하여 속도 향상 (선택)
- **일반 사용**: 브라우저 Web Worker만 사용 (현재 방식 유지)

## 🔧 현재 오류 해결 (선택 사항)

만약 WSL Proof Server를 사용하고 싶다면:

```bash
# WSL에서 npm install 실행
cd /mnt/e/zkp_final
npm install --legacy-peer-deps

# 또는 WSL 홈으로 복사 후 설치
cp -r /mnt/e/zkp_final ~/zkp_final
cd ~/zkp_final
npm install --legacy-peer-deps
```

하지만 **프론트엔드 코드도 수정해야** 실제로 사용할 수 있습니다.

## 📝 요약

**질문**: WSL Proof Server를 하는 이유가 뭘까?

**답변**: 
- **시연 시 속도 향상**을 위한 선택 사항
- 현재는 **브라우저 Web Worker만으로도 충분히 작동**
- **필수는 아니며**, 시연 환경에서만 필요할 수 있음
- 사용하지 않아도 시스템은 정상 작동

**결론**: **사용하지 않아도 됩니다.** 현재 브라우저 Web Worker 방식이 더 간단하고 충분합니다.
