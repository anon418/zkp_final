### 슬라이드 2: 기존 투표 시스템의 문제점

**제목**: "왜 기존 투표 시스템은 신뢰할 수 없을까?"

**내용**:

| 시스템 | 익명성 | 검증 가능성 | 조작 불가능성 |
|--------|--------|------------|--------------|
| Google Forms | ⚠️ IP 로그 남음 | ❌ 관리자만 확인 | ❌ 관리자 조작 가능 |
| 일반 투표 앱 | ⚠️ 서버 로그 | ❌ 믿어야 함 | ❌ 서버 조작 가능 |
| **VoteZK** | ✅ 완전 익명 | ✅ 누구나 검증 | ✅ 수학적 보장 |

**설명 포인트**:
1. **익명성 문제**: "IP 주소, 접속 시간이 서버에 남아 누가 투표했는지 알 수 있음"
2. **검증 불가능**: "관리자가 결과를 바꿔도 알 수 없음"
3. **조작 가능성**: "서버 관리자가 원하는 결과로 조작 가능"

**비유**:
- "투표함을 관리자가 직접 열어볼 수 있는 상황"
- "투표 결과를 관리자가 마음대로 바꿀 수 있는 상황"

### 슬라이드 3: 왜 ZKP가 필요한가?

**제목**: "Zero-Knowledge Proof란?"

**핵심 개념** (비유로 설명):
```
일반 투표:
"나는 A 후보를 선택했습니다" → 서버가 알 수 있음 ❌

ZKP 투표:
"나는 유효한 투표를 했고, 자격이 있으며, 중복 투표가 아닙니다"
→ 하지만 누구를 선택했는지는 알 수 없음 ✅
```

**3가지 핵심 보장**:
1. ✅ **자격 증명**: "투표할 권리가 있는 사람입니다"
2. ✅ **중복 방지**: "이미 투표한 사람은 다시 투표할 수 없습니다"
3. ✅ **익명성**: "누구를 선택했는지는 알 수 없습니다"

**설명 포인트**:
- "수학적으로 증명하되, 실제 선택 내용은 숨김"
- "블록체인에 기록되어 누구나 검증 가능"

---

### 슬라이드 4: VoteZK의 핵심 가치

**제목**: "VoteZK가 해결하는 것"

**3가지 핵심 가치**:

#### 🔒 1. 운영자를 믿지 않아도 됨
- 블록체인에 기록 → 수학적으로 조작 불가능
- Etherscan에서 누구나 직접 확인 가능

#### 🎭 2. 완전한 익명성
- Zero-Knowledge Proof로 서버도 모름
- IP 주소, 접속 시간 등 로그 없음

#### ⛓️ 3. 영구 검증 가능
- Sepolia 블록체인에 영구 기록
- 언제든지 재검증 가능

**비유**:
- "투명한 투표함 + 자물쇠가 수학적으로 잠겨있어서 조작 불가능"
- "투표 내용은 보이지 않지만, 유효성은 증명됨"

### 슬라이드 8: ZKP 증명 생성 과정

**제목**: "투표자가 ZKP 증명을 만드는 과정"

**핵심 과정** (15초 소요):

```
1. 후보 선택
   ↓
2. ZKP 증명 생성 (Web Worker)
   - nullifierSecret 생성
   - Merkle Proof 생성
   - Groth16 Proof 생성
   ↓
3. Public Signals 추출
   - Merkle Root
   - Poll ID
   - Nullifier (중복 방지)
   - Vote Commitment (암호화된 투표)
   ↓
4. Proof + Public Signals 준비 완료
```

**설명 포인트**:
- "15초 동안 1300개의 제약조건을 계산"
- "투표 내용은 Vote Commitment로 암호화되어 공개되지 않음"
- "Nullifier로 중복 투표 방지"

**비유**:
- "수학 문제를 풀어서 '나는 자격이 있고, 중복 투표가 아니야'를 증명"
- "하지만 실제로 누구를 선택했는지는 숨김"

### 슬라이드 9: 블록체인 검증

**제목**: "블록체인에서 검증하는 방법"

**검증 과정**:

```
1. Relayer가 Proof를 블록체인에 제출
   ↓
2. Groth16Verifier.verifyProof() 호출
   - Proof 유효성 검증
   - Public Signals 검증
   ↓
3. VotingV2.vote() 실행
   - Nullifier 중복 체크
   - Vote Commitment 저장
   ↓
4. 트랜잭션 완료
   - Etherscan에 기록
   - 누구나 확인 가능
```

**설명 포인트**:
- "스마트 컨트랙트가 수학적으로 Proof를 검증"
- "검증 성공 시에만 투표가 기록됨"
- "Etherscan에서 직접 확인 가능"

**시연 포인트**:
- Etherscan 링크 클릭하여 실제 트랜잭션 보여주기
- "VoteCast 이벤트" 확인

### 슬라이드 11: 익명성 보장 메커니즘

**제목**: "왜 익명성이 보장되는가?"

**3가지 보호 계층**:

#### 1. ZKP Proof
- 투표 내용이 Proof에 포함되지 않음
- Vote Commitment로 암호화

#### 2. Nullifier
- 중복 방지용 해시값
- 실제 신원과 연결 불가능

#### 3. 블록체인 기록
- 투표 내용 없이 유효성만 기록
- Etherscan에서도 선택 내용 확인 불가

**비유**:
- "투표함에 투표지를 넣지만, 투표지 내용은 보이지 않음"
- "하지만 '유효한 투표지'라는 것은 증명됨"
