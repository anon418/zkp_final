## 📖 용어집

### 핵심 개념

#### **Zero-Knowledge Proof (ZKP, 영지식 증명)**

- **정의**: 정보를 공개하지 않고도 그 정보가 유효함을 증명하는 암호학적 방법
- **예시**: "나는 유효한 투표를 했지만, 무엇을 선택했는지는 비밀"
- **기술**: Groth16 증명 시스템 사용
- **효과**: 투표 내용을 숨기면서도 유효성 검증 가능

#### **Nullifier**

- **정의**: 중복 투표를 방지하기 위한 고유 식별자
- **생성**: `Poseidon(nullifierSecret, pollId)` 해시 함수로 생성
- **특징**:
  - 같은 `nullifierSecret`과 `pollId` 조합이면 항상 같은 nullifier 생성
  - 역추적 불가능 (해시값이므로)
- **용도**: 같은 nullifier로 재투표 시 감지 및 차단

#### **Relayer (릴레이어)**

- **정의**: 사용자 대신 가스비를 지불하는 서버
- **역할**:
  - 사용자는 가스비 없이 투표 가능
  - Relayer가 Sepolia ETH로 트랜잭션 제출
- **장점**: 사용자 경험 개선 (가스비 걱정 없음)

#### **Public Signals (공개 신호)**

- **정의**: ZKP 증명에서 공개되는 정보
- **구성**: `[merkleRoot, pollId, nullifier, voteCommitment]`
- **특징**:
  - 투표 내용(후보 선택)은 포함되지 않음
  - 블록체인에서 검증 가능
  - 누구나 확인 가능하지만 신원은 보호됨

#### **Vote Commitment (투표 커밋먼트)**

- **정의**: 투표 내용을 암호화한 값
- **특징**:
  - 원본 투표 복원 불가능
  - 익명성 보장
  - 블록체인에 기록되어 검증 가능

### 블록체인 용어

#### **Sepolia Testnet**

- **정의**: 이더리움 테스트 네트워크
- **용도**: 개발 및 테스트용 (무료 ETH 사용 가능)
- **특징**: 실제 이더리움과 동일한 기능, 가치 없음

#### **Etherscan**

- **정의**: 블록체인 탐색기 (이더리움용)
- **용도**: 트랜잭션, 스마트 컨트랙트, 이벤트 확인
- **URL**: https://sepolia.etherscan.io

#### **Smart Contract (스마트 컨트랙트)**

- **정의**: 블록체인에 배포된 자동 실행 프로그램
- **VoteZK의 컨트랙트**:
  - `VotingV2`: 투표 로직 관리
  - `Groth16Verifier`: ZKP 증명 검증

### 기술 용어

#### **Circom**

- **정의**: ZKP 회로 설계 언어
- **용도**: 투표 증명 회로 작성
- **출력**: WASM 파일, ZKEY 파일

#### **Groth16**

- **정의**: ZKP 증명 시스템 중 하나
- **특징**:
  - 짧은 증명 크기
  - 빠른 검증 속도
  - 널리 사용됨

#### **Merkle Tree**

- **정의**: 데이터 무결성을 검증하는 트리 구조
- **용도**: 화이트리스트 검증 (현재는 Open Poll로 미사용)
- **특징**: 루트 해시만 공개해도 전체 목록 검증 가능

## ❓ 자주 묻는 질문

### Q: ZKP가 뭔가요?

**A**: Zero-Knowledge Proof는 정보를 공개하지 않고도 그 정보가 유효함을 증명하는 방법입니다. VoteZK에서는 투표 내용을 숨기면서도 유효한 투표임을 증명합니다.

### Q: 왜 블록체인을 사용하나요?

**A**: 블록체인에 기록하면 조작이 어렵고, 누구나 Etherscan에서 검증할 수 있습니다. 투명성과 신뢰성을 보장합니다.

### Q: 정말 익명인가요?

**A**: 네, ZKP를 사용하므로 서버도 누가 투표했는지 알 수 없습니다. 다만 같은 지갑 주소로는 중복 투표가 불가능합니다.

### Q: 가스비를 내야 하나요?

**A**: 아니요, Relayer가 가스비를 대납하므로 사용자는 가스비를 지불할 필요가 없습니다.
